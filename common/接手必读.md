## 2. 跨机部署踩坑表

### 2.1 Python 环境

| 包 | 备注 |
|---|---|
| `openctp-ctp>=6.7.11` | 预编译 wheel, **不需要 MSVC**, 推荐 |
| `requests` | 钉钉/Bark 推送用, 可选 (缺了降级 dryrun) |
| `fastapi` + `uvicorn` | 手机确认桥用, 可选 (做单元测试不需要) |
| `numpy` / `pandas` | 策略层用, 必须装 |
| ~~`ctpbee`~~ | **不要装**, MSVC 编译会失败, Python 3.14 也没 `distutils` 了 |

### 2.2 `ctpbee` 编译失败

- **症状**：`Microsoft Visual C++ 14.0 or greater is required`
- **原因**：ctpbee 是 C 扩展, Python 3.14 + Win11 默认不带 MSVC
- **解决**：直接用 `openctp-ctp`（预编译 wheel），不要尝试 ctpbee

### 2.3 Python 3.14 删了 `distutils`

- **症状**：`ModuleNotFoundError: No module named 'distutils'`
- **解决**：用 `setuptools` 替代；或降到 Python 3.12

### 2.4 SimNow 新版看穿式前置（2025/06/19 切换）

| 环境 | 地址 | 用途 |
|---|---|---|
| 第一套 | `182.254.243.31:30001` | 交易时段主用（推荐） |
| 第二套 | `182.254.243.31:30002` | 备选 |
| 第三套 | `182.254.243.31:30003` | 备选 |
| 7x24 | `182.254.243.31:40001` | 任何时间可连，**新账号要等 3 个交易日**才能用 |

> 旧前置 `180.168.146.187:10201` 已于 2025/06/19 停用。

**默认认证码（不用单独申请！）**
- AppID = `simnow_client_test`
- AuthCode = `0000000000000000`（**16 个 0**）
- SimNow 默认开启终端认证，标准用户**不需要单独申请 AppID**
